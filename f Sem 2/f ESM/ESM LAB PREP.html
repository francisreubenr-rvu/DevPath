Part 1: Foundation Concepts

1.1 Raspberry Pi Pico Hardware Overview

The Raspberry Pi Pico is a microcontroller board built around the RP2040 chip, featuring a dual-core ARM Cortex-M0+ processor. For these labs, you’re using the non-WiFi variant.

Key Hardware Specifications:

  • Processor: Dual-core ARM Cortex-M0+ @ 133 MHz
  • Memory: 264KB on-chip SRAM
  • GPIO Pins: 26 multi-function GPIO pins (GP0-GP25)
  • Built-in LED: Connected to GP25 (referenced as LED_BUILTIN)
  • Programming: Via USB using Arduino IDE or MicroPython

Pin Numbering System:

  • GPxx (e.g., GP4, GP16) refers to GPIO pin numbers
  • Physical pin numbers on the board differ from GPIO numbers
  • Always use GPIO numbers in your code

📌 Exam Tip: Know the difference between GPIO pin numbers and physical pin positions. The exam may ask you to identify which GPIO pins are used in a given circuit diagram.

1.2 GPIO Pin Control Fundamentals

GPIO (General Purpose Input/Output) pins can be configured as either inputs or outputs. For LED control, we configure them as outputs.

Three Essential Functions:

pinMode(pin, mode);        // Configure pin as INPUT or OUTPUT
digitalWrite(pin, state);  // Set pin HIGH (3.3V) or LOW (0V)
delay(milliseconds);       // Pause execution

Electrical Characteristics:

  • HIGH state: 3.3V (not 5V like Arduino!)
  • LOW state: 0V (GND)
  • Maximum current per pin: 12mA (source), 12mA (sink)
  • Use current-limiting resistors (typically 220Ω-330Ω) with LEDs

LED Connection:

Pico GPIO → [220Ω Resistor] → [LED Anode(+)] → [LED Cathode(-)] → GND

📌 Exam Tip: Remember that delay() is a blocking function — nothing else happens while the delay is active. This is important for understanding program behavior.

Part 2: LED Programming (Programs 1-9)

Program 1: Blink 2 Alternate LEDs (3s & 5s delay)

Question: Write a program to blink 2 alternate LEDs out of 4 LEDs with delay of 3 and 5 seconds respectively.

Concept: Group control — turning multiple pins ON/OFF together in patterns.

const int pins[] = {4, 6, 9, 22};  // 4 GPIO pins for 4 LEDs

void setup() {
  for (int p : pins) pinMode(p, OUTPUT);  // Set all as OUTPUT
}

void loop() {
  // Group A (first two) ON, Group B (last two) OFF
  digitalWrite(pins[0], HIGH); 
  digitalWrite(pins[1], HIGH);
  digitalWrite(pins[2], LOW);  
  digitalWrite(pins[3], LOW);
  delay(3000);  // 3 seconds
  
  // Group B ON, Group A OFF
  digitalWrite(pins[0], LOW);  
  digitalWrite(pins[1], LOW);
  digitalWrite(pins[2], HIGH); 
  digitalWrite(pins[3], HIGH);
  delay(5000);  // 5 seconds
}

Key Learning Points:

  • Arrays store multiple pin numbers for cleaner code
  • Range-based for loop (for (int p : pins)) iterates through all array elements
  • Alternating patterns require explicit HIGH/LOW states for each group

📌 Exam Tip: When the question says “2 alternate LEDs out of 4,” interpret it as two groups of 2 LEDs each, alternating between which group is ON.

Program 3: Binary Counter 1→15→1 with 2s Pause

Question: Display numbers in increasing and decreasing order from 1 to 15 in binary form using 4 LEDs, with 1 sec delay. When reaching max (15) or min (1), wait 2 seconds.

Binary Representation with 4 LEDs:

  • LED positions represent bit values: 2³ 2² 2¹ 2⁰ (MSB to LSB)
  • Number 5 = 0101 → LEDs show: OFF ON OFF ON
  • Number 12 = 1100 → LEDs show: ON ON OFF OFF
const int pins[] = {2, 3, 4, 5};  // LSB (2⁰) to MSB (2³)

void setup() {
  for (int p : pins) pinMode(p, OUTPUT);
}

void showBinary(int n) {
  for (int i = 0; i < 4; i++) {
    // Extract bit i from number n
    int bit = (n >> i) & 1;  // Right shift n by i, mask with 1
    digitalWrite(pins[i], bit);
  }
  delay(1000);
}

void loop() {
  // Count up 1 → 15
  for (int i = 1; i <= 15; i++) {
    showBinary(i);
    if (i == 15) delay(2000);  // Extra pause at max
  }
  
  // Count down 14 → 1
  for (int i = 14; i >= 1; i--) {
    showBinary(i);
    if (i == 1) delay(2000);  // Extra pause at min
  }
}

Understanding Bit Manipulation:

  • >> is right shift: 5 >> 1 = 2 (binary: 101 → 10)
  • & is bitwise AND: 5 & 1 = 1 (binary: 101 & 001 = 001)
  • Combined: (n >> i) & 1 extracts the i-th bit of n

Example for number 11 (binary 1011):

  • i=0: (11 >> 0) & 1 = 11 & 1 = 1 (bit 0 = 1) → LED 0 ON
  • i=1: (11 >> 1) & 1 = 5 & 1 = 1 (bit 1 = 1) → LED 1 ON
  • i=2: (11 >> 2) & 1 = 2 & 1 = 0 (bit 2 = 0) → LED 2 OFF
  • i=3: (11 >> 3) & 1 = 1 & 1 = 1 (bit 3 = 1) → LED 3 ON

📌 Exam Tip: Bit manipulation is crucial. Practice extracting individual bits using shift and mask operations. This pattern appears in many embedded systems questions.

Program 5: Serial Input Controls LED

Question: Listen for serial monitor input and turn ON one of four LEDs when a specific number (1-4) is entered.

const int pins[] = {4, 6, 9, 22};

void setup() {
  Serial.begin(9600);
  for (int p : pins) pinMode(p, OUTPUT);
  Serial.println("Enter 1-4 to control LEDs:");
}

void loop() {
  if (Serial.available() > 0) {
    int num = Serial.parseInt();  // Read integer from serial
    
    if (num >= 1 && num <= 4) {
      // Turn all OFF first
      for (int p : pins) digitalWrite(p, LOW);
      
      // Turn selected LED ON
      digitalWrite(pins[num - 1], HIGH);
      
      Serial.print("LED ");
      Serial.print(num);
      Serial.println(" is ON");
    } else {
      Serial.println("Invalid! Enter 1-4");
    }
  }
}

Key Concept: Serial.parseInt() reads characters until it gets a valid integer, then returns it.

Program 8: Chase Effect - Bidirectional (2s delay)

Question: Display blinking 4 LEDs from left to right and vice versa with a delay of 2 sec.

const int pins[] = {2, 3, 4, 5};

void setup() {
  for (int p : pins) pinMode(p, OUTPUT);
}

void loop() {
  // Left to right (0 → 3)
  for (int i = 0; i < 4; i++) {
    for (int p : pins) digitalWrite(p, LOW);
    digitalWrite(pins[i], HIGH);
    delay(2000);
  }
  
  // Right to left (2 → 0)
  for (int i = 2; i >= 0; i--) {
    for (int p : pins) digitalWrite(p, LOW);
    digitalWrite(pins[i], HIGH);
    delay(2000);
  }
}

Note: The second loop starts at index 2 (not 3) to avoid showing LED 3 twice in a row.

Part 3: ARM Assembly Programming (Programs 10-19)

3.1 ARM Cortex-M0+ Architecture Basics

The ARM Cortex-M0+ is a 32-bit RISC processor with a simplified instruction set designed for low-power embedded applications.

Key Features:

  • 32-bit registers: r0-r15
  • Thumb instruction set: 16-bit and 32-bit instructions
  • Load/Store architecture: Only LOAD and STORE instructions access memory
  • Simple pipeline: 2-stage pipeline (fetch, decode/execute)

Register Overview:

Register Name Purpose Preserved?
r0-r3 Argument/Scratch Function arguments & return values No
r4-r11 Variable General purpose Yes (callee-saved)
r12 IP Intra-procedure scratch No
r13 SP Stack Pointer Yes
r14 LR Link Register (return address) No
r15 PC Program Counter Special

Calling Convention:

  • Arguments: Pass in r0, r1, r2, r3 (overflow goes on stack)
  • Return value: Place in r0
  • Return instruction: bx lr (branch to link register)

3.2 Assembly File Structure

All assembly programs for Programs 10-19 follow this structure:

.global my_asm_fn    ; Make function visible to C
.text                ; Code section
my_asm_fn:
  ; Your code here
  bx lr              ; Return to caller
.end

C Wrapper (same for all):

#include <stdio.h>
extern "C" { unsigned int my_asm_fn(void); }

void setup() { 
  Serial.begin(9600); 
  pinMode(LED_BUILTIN, OUTPUT); 
}

void loop() {
  unsigned int result = my_asm_fn();
  Serial.print("Result: ");
  Serial.println(result);
  
  digitalWrite(LED_BUILTIN, HIGH); delay(1000);
  digitalWrite(LED_BUILTIN, LOW);  delay(1000);
}

Program 10: Return Constant 200 / Sum of Three Constants

Question 10a: Write an assembly function that returns 200 back to the C program.

.global my_asm_fn
.text
my_asm_fn:
  mov r0, #200    ; Load 200 into r0 (return register)
  bx  lr          ; Return to caller
.end

Explanation:

  • mov r0, #200 — Move immediate value 200 into register r0
  • # prefix indicates immediate value (constant)
  • r0 is the return register by ARM calling convention
  • bx lr branches to the address in the link register (returns to caller)

Question 10b: Return sum of three constants 20, 25, and 40.

.global my_asm_fn
.text
my_asm_fn:
  mov r1, #20     ; Load 20 into r1
  mov r2, #25     ; Load 25 into r2
  mov r3, #40     ; Load 40 into r3
  add r0, r1, r2  ; r0 = r1 + r2 = 20 + 25 = 45
  add r0, r0, r3  ; r0 = r0 + r3 = 45 + 40 = 85
  bx  lr          ; Return r0 (85)
.end

ADD Instruction Format:

  • add rd, rn, operand2 — rd = rn + operand2
  • operand2 can be a register or immediate value

Program 11: Sum of Four Constants (10 + 15 + 25 + 35 = 85)

my_asm_fn:
  mov r1, #10

  mov r3, #25

  add r0, r1, r2   ; r0 = 10 + 15 = 25
  add r0, r0, r3   ; r0 = 25 + 25 = 50
  add r0, r0, r4   ; r0 = 50 + 35 = 85

.end

Program 12: Sum of Five Constants (10 + 20 + 30 + 40 + 50 = 150)

.global my_asm_fn
.text
my_asm_fn:
  mov r1, #10
  mov r2, #20
  mov r3, #30
  mov r4, #40
  mov r5, #50
  add r0, r1, r2   ; r0 = 10 + 20 = 30
  add r0, r0, r3   ; r0 = 30 + 30 = 60

Program 13: Add Three Input Parameters from C

Question: Take three constants as inputs from C program, perform ADD operation, return result.

C Code:

extern "C" { unsigned int my_asm_fn(unsigned int a, unsigned int b, unsigned int c); }

void loop() {

my_asm_fn:

.end

Program 14: AND & OR Operations on Two Parameters

Question: Take two input parameters, perform AND and OR logic operations, return result.

Interpretation: Since we can only return one value, let’s assume the question wants AND result.

C Code:

void loop() {
  unsigned int result = my_asm_fn(0b1100, 0b1010);

Assembly (AND version):

.global my_asm_fn
.text
my_asm_fn:
  ; Inputs: r0=a, r1=b
  and r0, r0, r1   ; r0 = a & b
  bx  lr
.end

Assembly (OR version):

.global my_asm_fn
.text
my_asm_fn:

.end

Program 16: Logical Shift Right 5 Times

Question: Perform 5 times logical shift right operation on one parameter.

Logical Shift Right (LSR):

  • Shifts bits to the right
  • Fills vacated bits on the left with 0
  • Example: 0b11010000 LSR 5 = 0b00000110
.global my_asm_fn
.text
my_asm_fn:
  ; Input: r0
  lsr r0, r0, #5   ; Logical shift right by 5 positions
  bx  lr
.end

Alternative (shift 5 times individually):

my_asm_fn:
  lsr r0, r0, #1
  lsr r0, r0, #1
  lsr r0, r0, #1
  lsr r0, r0, #1
  lsr r0, r0, #1
  bx  lr

Example:

  • Input: 160 (0b10100000)
  • After LSR 5: 5 (0b00000101)

Program 17: Logical Shift Left 4 Times

Question: Perform 4 times logical shift left operation on one parameter.

.global my_asm_fn
.text
my_asm_fn:
  ; Input: r0
  lsl r0, r0, #4   ; Logical shift left by 4 positions
  bx  lr
.end

Logical Shift Left (LSL):

  • Shifts bits to the left
  • Fills vacated bits on the right with 0
  • Example: 0b00001010 LSL 4 = 0b10100000

Example:

  • Input: 10 (0b00001010)
  • After LSL 4: 160 (0b10100000)
  • Mathematical effect: multiplication by 2⁴ = 16

Program 18: LSR 2 Times, Then LSL 4 Times

Question: Perform 2 times logical shift right and 4 times logical shift left on one parameter.

.global my_asm_fn
.text
my_asm_fn:
  ; Input: r0
  lsr r0, r0, #2   ; Shift right by 2
  lsl r0, r0, #4   ; Shift left by 4
  bx  lr
.end

Net Effect Analysis:

  • LSR 2 divides by 4
  • LSL 4 multiplies by 16
  • Net: multiply by 4 (with loss of 2 LSBs from the initial LSR)

Example:

  • Input: 25 (0b00011001)
  • After LSR 2: 6 (0b00000110) — lost bits 01
  • After LSL 4: 96 (0b01100000)

Program 19: Arithmetic Shift Right 8 Times on Signed Number

Question: Perform 8 times arithmetic shift right operation on one signed (negative) parameter.

Arithmetic Shift Right (ASR):

  • Like LSR, but preserves sign bit
  • Fills vacated bits with the sign bit (bit 31)
  • Used for signed division by powers of 2
.global my_asm_fn
.text
my_asm_fn:
  ; Input: r0 (signed)
  asr r0, r0, #8   ; Arithmetic shift right by 8
  bx  lr
.end

Example with Negative Number:

  • Input: -256 (0xFFFFFF00 in 32-bit two’s complement)
  • After ASR 8: -1 (0xFFFFFFFF)
  • Sign bit preserved throughout

Difference from LSR:

LSR: 0xFFFFFF00 >> 8 = 0x00FFFFFF (positive, wrong!)
ASR: 0xFFFFFF00 >> 8 = 0xFFFFFFFF (negative, correct!)

📌 Exam Tip: Know the difference between LSR (logical) and ASR (arithmetic). LSR is for unsigned, ASR is for signed numbers. Exam questions will test this distinction.

Program 20: De Morgan’s Theorem Verification

Question: Verify De Morgan’s theorem with two inputs.

De Morgan’s Theorems:

  1. NOT (A AND B) = (NOT A) OR (NOT B)
  2. NOT (A OR B) = (NOT A) AND (NOT B)

Assembly (Theorem 1):

.global my_asm_fn
.text
my_asm_fn:
  ; Inputs: r0=A, r1=B
  ; Calculate LHS: NOT(A AND B)
  and r2, r0, r1   ; r2 = A AND B
  mvn r2, r2       ; r2 = NOT(r2) = NOT(A AND B)
  
  ; Calculate RHS: (NOT A) OR (NOT B)
  mvn r3, r0       ; r3 = NOT A
  mvn r4, r1       ; r4 = NOT B
  orr r3, r3, r4   ; r3 = (NOT A) OR (NOT B)
  
  ; Compare: if r2 == r3, theorem verified
  cmp r2, r3
  mov r0, #0       ; Assume false
  moveq r0, #1     ; Set to 1 if equal
  bx  lr
.end

C Verification:

void loop() {
  unsigned int a = 0b1100;
  unsigned int b = 0b1010;
  unsigned int result = my_asm_fn(a, b);
  
  if (result == 1) {
    Serial.println("De Morgan's theorem is VERIFIED");
  } else {
    Serial.println("De Morgan's theorem is NOT verified");
  }
  
  delay(3000);
}

📌 Exam Tip: The exam may ask for either theorem 1 or 2, or both. Be prepared to implement both in assembly.

Program 23: Read Flag Bits from PSR

Question: Read the flag bits from the Program Status Register (PSR). Return both outputs to main function.

PSR Flag Bits:

  • N (Negative): Bit 31 — Result was negative
  • Z (Zero): Bit 30 — Result was zero
  • C (Carry): Bit 29 — Carry out occurred
  • V (Overflow): Bit 28 — Signed overflow occurred

Assembly:

.global my_asm_fn
.text
my_asm_fn:
  mrs r0, apsr     ; Move special register APSR to r0
  bx  lr
.end

C Code to Extract Flags:

void loop() {
  unsigned int psr = my_asm_fn();
  
  int N = (psr >> 31) & 1;  // Negative flag
  int Z = (psr >> 30) & 1;  // Zero flag
  int C = (psr >> 29) & 1;  // Carry flag
  int V = (psr >> 28) & 1;  // Overflow flag
  
  Serial.print("N="); Serial.print(N);
  Serial.print(" Z="); Serial.print(Z);
  Serial.print(" C="); Serial.print(C);
  Serial.print(" V="); Serial.println(V);
  
  delay(2000);
}

MRS Instruction: “Move from special Register to general register”

Program 24: CMP with EQ Condition

Question: Take two parameters and perform CMP with EQ condition code. Return 1 if equal, else 0.

Assembly:

.global my_asm_fn
.text
my_asm_fn:
  ; Inputs: r0=A, r1=B
  cmp r0, r1       ; Compare A and B (sets flags)
  mov r0, #0       ; Assume not equal
  moveq r0, #1     ; If equal, set r0 = 1
  bx  lr
.end

CMP Instruction: Performs subtraction (A - B) and updates flags, discards result.

Conditional Execution:

  • moveq r0, #1 — Execute MOV only if EQ flag is set (i.e., if A == B)

C Code:

void loop() {
  unsigned int a = 42;
  unsigned int b = 42;
  unsigned int result = my_asm_fn(a, b);
  
  if (result == 1) {
    Serial.println("The values are EQUAL");
  } else {
    Serial.println("The values are NOT equal");
  }
  
  delay(2000);
}

Program 25: LO (Unsigned Lower) Condition

Question: Check for LO (unsigned lower) condition. Return 1 if first parameter < second, else 0.

.global my_asm_fn
.text
my_asm_fn:
  ; Inputs: r0=A, r1=B

Common Exam Mistakes to Avoid

  1. Confusing GPIO numbers with physical pin numbers — Always use GPIO numbers (GPxx) in code

  2. Forgetting pinMode() — LEDs won’t turn on/off without setting pin mode to OUTPUT

  3. Wrong array indexing — Arrays are zero-indexed; pins[0] is the first element, not pins[1]

  4. Bit extraction errors — (n >> i) & 1 extracts bit i; don’t forget the & 1 mask!

  5. Assembly return register — Always place return value in r0, not r1 or other registers

  6. Missing bx lr — Your assembly function won’t return without this instruction

  7. OR vs ORR — In ARM assembly, it’s orr not or

  8. Signed vs unsigned shifts — Use ASR for signed, LSR for unsigned

  9. Condition code confusion — LO is for unsigned lower, not signed less than (use LT for that)

  10. Serial.parseInt() blocks — It waits for input; check Serial.available() first in production code

Study Tips for Lab Exam Success

1. Hands-On Practice

  • Type out each program yourself, don’t just copy-paste
  • Run every program on your Pico
  • Modify programs slightly to test your understanding

2. Understand, Don’t Memorize

  • Know WHY each line of code is needed
  • Be able to explain the logic flow to someone else
  • Trace through programs mentally with example values

3. Pattern Recognition

  • Notice common patterns (array iteration, bit extraction, assembly templates)
  • Group similar programs together in your mind
  • Use the existing code as templates for variations

4. Time Management

  • Practice writing complete programs within 10-15 minutes
  • Know which programs are quick (simple LED blink) vs complex (assembly logic)
  • Have your wiring setup memorized to save time

5. Common Variations to Prepare

  • Different GPIO pin numbers (the exam may change these)
  • Different delay times (3s, 5s, 10s, etc.)
  • Different constants in assembly (20, 50, 100 instead of 10, 20, 30)
  • Different binary patterns (even/odd, multiples of 4, etc.)

6. Pre-Exam Checklist

  • Verify your Raspberry Pi Pico is working
  • Test all GPIO pins you might use
  • Have spare LEDs and resistors
  • Know how to recover a frozen Pico (hold BOOTSEL, reconnect USB)
  • Have your breadboard layout planned

Summary / TL;DR

LED Programming (1-9):

  • Use arrays to store pin numbers
  • Range-based for loops for cleaner iteration
  • Bit manipulation (n >> i) & 1 extracts individual bits for binary display
  • Chase effects use sequential ON/OFF patterns
  • Serial.parseInt() reads numbers from serial monitor

ARM Assembly (10-19):

  • Always return value in r0 register
  • Parameters arrive in r0-r3 automatically
  • bx lr returns to C caller
  • MOV loads values, ADD sums them
  • Logical operations: AND, ORR (not OR!), EOR (XOR), MVN (NOT)
  • Shifts: LSL (left), LSR (right unsigned), ASR (right signed)
  • Shift by N is equivalent to multiply/divide by 2^N

Logic Verification (20-30):

  • CMP subtracts and sets flags, discards result
  • Conditional execution: moveq, movlo, movhi, etc.
  • MRS reads special registers like APSR (flag bits)
  • De Morgan: NOT(A AND B) = (NOT A) OR (NOT B)
  • Know all truth tables by heart
  • Assembly returns 1 for verified, 0 for not verified

Key Takeaway: Every program follows predictable patterns. Master the patterns, and you can handle any variation the exam throws at you.
