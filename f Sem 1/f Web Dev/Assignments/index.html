The Collection

Excalibur

The legendary sword of King Arthur, said to have been given to him by the Lady of the Lake. It is associated with the rightful sovereignty of Great Britain.

Mjolnir

The mighty hammer of Thor, the Norse god of thunder. Forged by dwarves, it is capable of leveling mountains and always returns to its owner's hand when thrown.

Kusanagi-no-Tsurugi

The "Grass-Cutting Sword," one of the Three Imperial Regalia of Japan. According to legend, it was found in the body of an eight-headed serpent.

Gungnir

The magical spear of the Norse god Odin. It is said to have been crafted by dwarves and is inscribed with runes, never missing its target.

Durandal

The sword of Roland, a legendary paladin of Charlemagne. It was claimed to be indestructible and contained sacred Christian relics within its golden hilt.

Longclaw

A Valyrian steel bastard sword from "Game of Thrones." Originally the ancestral weapon of House Mormont, it was gifted to Jon Snow.

Master Sword

Also known as the "Blade of Evil's Bane" from "The Legend of Zelda" series. A sacred blade which can only be wielded by a hero with great courage.

Andúril

The "Flame of the West," a sword from "The Lord of the Rings." It was reforged from the shards of Narsil and wielded by Aragorn, King of Gondor.

Masamune

Blades created by Japan's greatest swordsmith, Gorō Nyūdō Masamune. His swords are legendary for their superior quality and beauty, often considered national treasures.

Aegis

The divine shield from Greek mythology, wielded by both Zeus and Athena. It bore the head of Medusa and was said to make a sound like a roar of a hundred dragons.

Web Concepts

DNS Resolution

When you enter a domain name (e.g., www.example.com), your computer uses the Domain Name System (DNS) to find its IP address (e.g., 192.0.2.1).

  1. Browser checks its local cache.
  2. If not found, it queries a DNS resolver (from your ISP).
  3. The resolver asks the root DNS server, which points to the Top-Level Domain (TLD) server (.com).
  4. The TLD server points to the Authoritative Name Server for the domain, which returns the IP address.
  5. The browser uses this IP to send an HTTP request to the web server.
  6. The server responds with the webpage.

Parts of a Domain Name

A domain name typically has three parts. For example, in www.google.com:

  • www: Subdomain (identifies a specific section).
  • google: Second-Level Domain (SLD) (the unique name).
  • .com: Top-Level Domain (TLD) (the category).

HTTP vs. HTTPS

HTTP (HyperText Transfer Protocol) transfers data in plain text, which can be intercepted.

HTTPS (HyperText Transfer Protocol Secure) uses SSL/TLS encryption to ensure data is transmitted securely. It protects user data, builds trust, and is essential for any site with logins or payments.

Relative vs. Absolute URLs

A Relative URL gives the path to a file relative to the current page's location, without the full domain.

Example: If you are on `.../about/index.html` and want to link to `.../about/contact.html`, you would use:

<a href="contact.html">Contact Us</a>

Use relative URLs for linking within the same website. This makes it easy to move the site to a new domain without breaking all your internal links.

HTML Tag Guides

Semantic HTML Tags

Semantic tags improve structure, accessibility, and SEO by clearly defining the role of content for browsers and assistive technologies.

Tag Purpose
<header> Defines the introductory content or header for a section or page.
<nav> Contains the main navigation links on a page or section.
<main> Represents the primary content of the document, unique to the page.
<section> Groups thematically related content within a page.
<article> Encapsulates a self-contained piece of content that can stand alone.
<aside> Defines content aside from the main content, such as sidebars.
<footer> Specifies footer content for a section or entire page.
<figure> Groups media content like images, videos, or diagrams.
<figcaption> Provides a caption or description for the <figure> element.
<mark> Highlights or marks text relevant to the current context.
<time> Represents a specific time or date in a machine-readable format.
<em> Marks emphasized text, often read with vocal emphasis by screen readers.
<strong> Indicates text of strong importance, usually styled bold.
<details> Represents additional details that users can view or hide.
<summary> Defines a visible heading for the <details> element.

Non-Semantic HTML Tags

Non-semantic tags are for visual presentation, layout, or grouping. They do not describe the content's purpose but are vital for styling and scripting.

Tag Purpose
<div> A generic block-level container used to group other elements for styling or layout.
<span> A generic inline container used to apply styles to a part of text.
<b> Bold text, used for stylistic purposes without implying extra importance.
<i> Italic text, used for stylistic offset like foreign words or technical terms.
<br> Inserts a line break; used purely for visual formatting.
<hr> Represents a thematic break or horizontal rule for visual separation.

CSS Display Property

The `display` property defines whether an element behaves as a block, inline, flex container, grid container, or other display types.

Value Description
block Element takes full line width, starts on a new line. (e.g., <div>, <p>)
inline Element flows inline with text and only takes necessary width. (e.g., <span>, <a>)
inline-block Behaves like inline but allows setting width, height, padding, and margin.
none Element is hidden from the layout and does not take any space.
flex Element behaves as a flexible box container, useful for responsive layouts.
grid Element behaves as a grid container, allowing 2D layout control.

About The Armoury

From the divine artifacts of gods to the fabled blades of epic heroes, this collection serves as a tribute to the stories that have shaped cultures and inspired imaginations for centuries.
